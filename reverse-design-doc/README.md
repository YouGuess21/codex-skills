# Reverse Design Doc

Reverse-engineer a `rev-design.md` from an existing codebase.

Use this when the implementation exists but the design is undocumented, or when you want a reusable architecture artifact for future sessions and contributors.

Typical input:

- a codebase
- optionally a subsystem or feature area to focus on

Typical output:

- a `rev-design.md` covering system purpose, major components, flows, seams, and risks

This skill is grounded in code inspection rather than speculation.
