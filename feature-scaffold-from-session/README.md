# Feature Scaffold From Session

Turn prior Codex sessions into a fresh feature workspace scaffold.

Use this when a feature has already been explored in past sessions and you want a clean starting directory for the next cycle of work.

Typical input:

- one or more session IDs
- a target output directory

Typical output:

- `AGENTS.md`
- `feature.md`
- a targeted `.codex/` control plane

This skill is focused on the initial scaffold only and does not generate later planning artifacts unless asked.
