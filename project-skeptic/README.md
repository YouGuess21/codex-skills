# Project Skeptic

Apply deliberate skepticism to a project, feature, plan, or architecture.

Use this when you want assumptions challenged early, scope questioned, and simpler alternatives surfaced before spending more engineering time.

Typical input:

- a project idea
- a design or requirements doc
- an implementation plan
- a codebase slice

Typical output:

- prioritized objections
- open questions
- kill criteria
- the smallest next validating experiment
